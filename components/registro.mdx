# Registration Component

The registration component provides a comprehensive user signup interface with four input fields: name, email, phone number, and password. It features a centered card design with green accent colors to differentiate from the login page.

## Overview

The registration page is built with HTML5 and styled using Tailwind CSS via CDN. It implements a POST form with four main input fields and includes navigation back to the login page.

## Form Submission

<Note>
  The form is currently configured with `action="#"` and `method="POST"`. In a production environment, you should replace the `#` with your actual registration endpoint and implement proper validation.
</Note>

```html
<form action="#" method="POST" class="space-y-4">
```

**Attributes:**
- `action="#"` - Placeholder for backend endpoint URL
- `method="POST"` - HTTP POST method for secure data transmission
- `class="space-y-4"` - Vertical spacing of 1rem (16px) between form elements

## Validation Considerations

<Warning>
  This implementation relies on HTML5 browser validation only. For production use, implement both client-side and server-side validation:
  - Name: Minimum length, character restrictions
  - Email: Format validation, uniqueness check
  - Phone: Format validation, optional regional format
  - Password: Strength requirements (length, complexity)
</Warning>

### Current HTML5 Validation
- **Email field:** Browser validates email format automatically
- **Other fields:** No explicit validation (consider adding `required` attribute)

### Recommended Enhancements
```html
<!-- Add required attribute and pattern validation -->
<input type="email" id="email" name="email" required 
       placeholder="ejemplo@correo.com"
       class="w-full px-4 py-2 border rounded-lg focus:outline-none focus:ring-2 focus:ring-green-500">

<!-- Add password requirements -->
<input type="password" id="password" name="password" required
       minlength="8" 
       placeholder="Ingresa tu contraseña"
       class="w-full px-4 py-2 border rounded-lg focus:outline-none focus:ring-2 focus:ring-green-500">
```

## Design Principles

### Color Scheme
- **Primary:** Green (`green-500`, `green-600`) for buttons and focus states
- **Secondary:** Blue (`blue-500`) for navigation links
- **Background:** Light gray (`gray-100`) for page, white for card
- **Text:** Dark gray (`gray-700`) for labels, medium gray (`gray-600`) for secondary text

### Why Green for Registration?
The green color scheme differentiates the registration page from the login page (which uses blue), providing visual context to users about which page they're on. Green also psychologically suggests "go" or "start," which aligns with creating a new account.

### Spacing System
- Consistent padding and margins using Tailwind's spacing scale
- Form spacing: `space-y-4` (1rem between fields)
- Card padding: `p-8` (2rem)
- Input padding: `px-4 py-2` (1rem horizontal, 0.5rem vertical)
- Top margin on navigation: `mt-4` (1rem)

## Integration Notes

When integrating this component with a backend:

1. **Update form action:** Replace `action="#"` with your registration endpoint
2. **Add CSRF protection:** Include CSRF tokens for security
3. **Implement validation:** Add JavaScript validation before submission
4. **Handle errors:** Display validation errors inline near each field
5. **Success redirect:** Redirect to login or dashboard after successful registration
6. **Loading states:** Add loading indicator during form submission

<Note>
  Consider implementing password confirmation by adding a second password field to ensure users enter their intended password correctly.
</Note>
